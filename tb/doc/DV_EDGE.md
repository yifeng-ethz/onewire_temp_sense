# onewire_master_controller DV — Edge / Boundary Cases

**Companion docs:** [DV_PLAN.md](DV_PLAN.md), [DV_HARNESS.md](DV_HARNESS.md), [DV_BASIC.md](DV_BASIC.md), [DV_PROF.md](DV_PROF.md), [DV_ERROR.md](DV_ERROR.md), [DV_CROSS.md](DV_CROSS.md), [DV_REPORT.md](DV_REPORT.md), [BUG_HISTORY.md](BUG_HISTORY.md)
**Parent:** [DV_PLAN.md](DV_PLAN.md)
**ID Range:** `E001-E130`
**Total:** 130 cases (0 implemented / 0 waived)

This document expands the EDGE bucket of `DV_PLAN.md` into 130 directed boundary cases that pin every numerical, timing, FIFO depth, ticket-lock arbitration, controller spine, IPC handshake, and Read-ROM probe corner of the `onewire_master_controller` (+ `onewire_master`) IP path. Every case fixes the boundary value in numbers and lists the assertion or scoreboard observation that constitutes pass. The cases are grouped by RTL subsystem so that a regression failure isolates to the right module on inspection. No EDGE cases are implemented yet; all rows start at status "planned" via the parent DV_PLAN.

**Methodology key:**
- **D** = Directed (hand-crafted boundary stimulus, deterministic)
- **R** = Constrained-random (LCG-based PRNG; full Mentor license enables SV `rand` and constrained-random)
